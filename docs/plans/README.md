# Implementation plans

Files in this directory are historical execution plans. They explain how a
specific version was built, but they are not current architecture contracts.

For current authority, use this order:

1. [`AGENTS.md`](../../AGENTS.md)
2. accepted [`docs/adr/`](../adr/README.md) records
3. [`DESIGN.md`](../../DESIGN.md)
4. [`PHILOSOPHY.md`](../../PHILOSOPHY.md)

Completed plans stay in Git when they provide useful implementation history.
Outdated requirements should be corrected in canonical docs and Issues rather
than silently rewriting the historical plan.
